Configuration overview

The task of configuring the add-on must be performed by a user with administrative privileges. As shown in the image below, basic configuration entails:

  • setting search criteria for a data model. This includes options such as search sensitivity and permissions. The criteria you specify applies to all tables included in this data model.

  • determining which tables users can search. Because of the model-driven configuration, you only need to setup a table one time. Users can run a search on the table regardless of its dataset location.

  • indicating the field(s) included in a search operation. This includes specifying the field, and search algorithm applied to queries on this field.

In addition to the previously mentioned options, you can also configure:

  • Synonyms used to return results when a query does not find an exact match.

  • Expose add-on functionality as a webservice.

  • Configure your own search algorithm.

  • Set values to be excluded from a search.

  • Set up search history.
